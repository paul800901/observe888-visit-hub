## 結論

1. 不需要把策略改成追求泛流量；長期最有價值的仍是 `台南整復`、`台南整骨`、`台南整復推拿`、費用與店別相關查詢。
2. 服務頁與 FAQ 的方向正確；前 3 個月與全期間資料都支持補強搜尋語言、費用與安全邊界。
3. 舊 `.php` URL 是比「再寫更多內容」更大的技術清理項。GitHub Pages 目前無法直接做 server-side 301；若有 Cloudflare、DNS/CDN、原主機或網路公司可設定轉址，應優先處理。
4. 不建議大量新增 `台南推拿`、`按摩店`、`筋膜放鬆課程` 這類薄頁，避免把品牌定位拉向一般按摩或課程流量。
